관리자 페이지는 이제 **별도의 ID와 비밀번호**로 접근합니다.

```
아이디: admin
```

## 📋 로그인 절차

### 1단계: 데이터 초기화 (최초 1회)
```
/init-data 페이지 접속
  ↓
"데이터베이스 초기화 실행" 버튼 클릭
  ↓
✅ "21명의 직원 데이터가 등록되었습니다!" 메시지 확인
```

### 2단계: 관리자 로그인
```
/admin/login 페이지 접속
  ↓
아이디: admin
비밀번호: hankyung2024!
  ↓
"로그인" 버튼 클릭
  ↓
✅ "/admin" 페이지로 자동 이동
```

## 🚪 접근 시나리오

### **시나리오 1: 처음 접속하는 경우**
```
1. /init-data → 데이터 초기화 (21명)
2. /admin/login → 관리자 로그인
3. /admin → 관리자 대시보드 접근
```

### **시나리오 2: 이미 로그인한 경우**
```
1. /admin 직접 접속
2. ✅ 세션 확인 → 즉시 대시보드 표시
```

### **시나리오 3: 로그아웃 후**
```
1. /admin → "관리자 로그인이 필요합니다"
2. /admin/login으로 자동 리다이렉트
3. 다시 로그인
```

## 🚨 문제 해결

### **"관리자 로그인이 필요합니다" 메시지**
- 원인: 로그인하지 않았거나 세션이 만료됨
- 해결: `/admin/login`에서 다시 로그인

### **"아이디 또는 비밀번호가 올바르지 않습니다"**
- 원인: 잘못된 계정 정보 입력
- 해결: 
  - 아이디: `admin` (정확히 입력)
  - 비밀번호: `hankyung2024!` (대소문자 구분)

### **"인증 정보가 올바르지 않습니다"**
- 원인: localStorage의 데이터가 손상됨
- 해결: 
  1. 브라우저 개발자 도구 → Application → Local Storage
  2. `adminAuth` 삭제
  3. 다시 로그인

## ⚙️ 관리자 비밀번호 변경 방법

현재 비밀번호는 `/src/app/pages/AdminLoginPage.tsx` 파일에 하드코딩되어 있습니다.

### **비밀번호 변경 단계:**

1. `/src/app/pages/AdminLoginPage.tsx` 파일 열기
2. 다음 부분 수정:
```typescript
const ADMIN_USERNAME = "admin";  // 원하는 아이디
const ADMIN_PASSWORD = "hankyung2024!";  // 원하는 비밀번호
```
3. 저장 후 재배포

⚠️ **권장사항**: 
- 실제 운영 환경에서는 환경 변수 사용
- 비밀번호 최소 12자 이상
- 특수문자, 숫자, 대소문자 조합
